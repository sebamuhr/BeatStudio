# Beatbox to MIDI — Progress

> **PIVOT (2026-07-01): going native.** The browser build kept stuttering (full-canvas
> repaints) and the user decided it should be a **native Linux desktop app + phone
> companion**, not a browser app. New native app lives in **`desktop/`** (Python + PySide6
> GPU timeline, sounddevice audio, local-AI sound matching). The web app (`Beatbox to
> MIDI.dc.html`, currently v0.7.6) stays as the **phone capture companion** that syncs to
> the desktop via `server.py`. First native milestone done: window + toolbar + ruler +
> track headers + GPU-culled timeline rendering, matched to the web look (see
> `desktop/README.md`). Next: audio engine, then port extraction/matching + settings/editing.

**Web build version: v0.7.6** (title bar + `sw.js` cache name kept in sync; bump together).

Record a beatbox groove through the mic → it auto-splits each distinct sound into its
own instrument track → edit on a timeline (tempo / quantize / metronome / EQ / per-track
gate) → play back → export `.mid`. Plus: install as an app, work offline, and record on
your phone and have it show up on your computer.

## How to run

The app is **served by `server.py`** (which also provides the sync API).

- **On the laptop (quick test, no cert):**
  ```
  python3 server.py --http --port 8080
  ```
  Open `http://localhost:8080/Beatbox%20to%20MIDI.dc.html`.
  `localhost` is already a "secure context", so the mic works and there's **no cert warning**.

- **On the phone (same WiFi):**
  ```
  ./make-cert.sh          # once (re-run if the laptop's LAN IP changes)
  python3 server.py       # HTTPS on :8443
  ```
  Trust `certs/rootCA.pem` on the phone once
  (Settings → Security → Encryption & credentials → Install a certificate → CA certificate),
  then open `https://<laptop-LAN-IP>:8443/Beatbox%20to%20MIDI.dc.html` and "Add to Home screen".
  The mic needs HTTPS because the phone reaches the laptop over a LAN IP (not localhost).

> After editing the HTML/SW, **reload the installed PWA twice** (first reload activates the
> new service worker, second serves it). Confirm the title bar shows the new version.

## Architecture (non-obvious)

- `Beatbox to MIDI.dc.html` is a **component for a hosted "DC" runtime** (`support.js`,
  built from `dc-runtime`). Template syntax: `{{ }}`, `<sc-if>`, `<sc-for>`,
  `class Component extends DCLogic`.
- The DC runtime keeps the **logic instance separate from React**: a React host component
  holds it as `host.logic` (with `.state`, `.setState`, `.onPlay`, …). React only renders
  the object returned by `renderVals()`.
- It normally needs `React`/`ReactDOM` injected by the platform. We **vendored React 18 UMD**
  into `vendor/` and load it (guarded) in `<head>`, so the app **boots standalone / offline**.
- Audio: Web Audio API. Mic capture uses an **AudioWorklet** (`dc-capture`, loaded from a Blob
  URL in `_ensureWorklet`, node built in `_makeCaptureNode`) — replaced the deprecated
  `ScriptProcessorNode`. AudioWorklet + service worker + mic all require a secure context.

### Key files
| File | Purpose |
|------|---------|
| `Beatbox to MIDI.dc.html` | The whole app (template + logic in one `<script>`). |
| `support.js` | The DC runtime (do not edit). |
| `server.py` | HTTPS/HTTP server + sync API (`/api/ping|projects`). |
| `make-cert.sh` | Local CA + LAN cert for phone HTTPS. |
| `sw.js` | Service worker (offline + installable). Version = app version. |
| `manifest.webmanifest`, `icon-*`, `favicon-32`, `apple-touch-icon` | PWA install assets. |
| `vendor/react*.js` | Vendored React 18 so the app boots without the DC platform. |
| `certs/` | Generated by `make-cert.sh` (rootCA.pem → trust on phone). |
| `synced/` | Grooves uploaded from the phone land here as `<id>.json`. |
| `dev-test.mjs` | Headless smoke/loop test (see "Testing"). |

## Phone → computer sync (offline-first, no tunnel)

Goal: beatbox in the park offline, save it, and have it appear on the laptop once back on
the same WiFi. Saves include **both the detected beats AND the original audio** (16 kHz mono),
because auto-detection isn't perfect and you re-tweak on the computer.

- Because the app is served *by* `server.py`, the sync API is **same-origin** — `fetch('/api/...')`
  needs no IP config.
- **Save** (toolbar) → `onSaveGroove` stores the groove in **IndexedDB** (store `grooves`),
  marked `synced:false`. Works fully offline.
- **Auto-sync** (`_syncNow`): on mount, every 15 s, and on the `online` event — pings `/api/ping`;
  if reachable, POSTs unsynced grooves (audio base64 via `_b64FromF32`) to `/api/projects`,
  marks them synced. Uploaded grooves land in `synced/` and list via `/api/projects`.
- **Grooves** (toolbar, with online dot) → modal listing grooves with status
  (On this device / Synced ✓ / On computer) and **Open** to load one back (`_loadGroove`,
  rebuilds `_audioBuf`).
- `sw.js` **bypasses `/api/`** (never cached) so the app can't fake "online" or show a stale list.

## Done

  **Changing the instrument keeps the beats on the grid** — only `sound`+`type` change, events
  are untouched (no per-type stashing). Removed the old "+ Synth track" button and
  "Make a Synth/Drum" toggle (`_toggleType` left dead in place).
- **Master settings panel layout** fixed: grid columns bound to `gridCols` (`1fr` for master,
  `240px 1fr` otherwise) so the master recording area isn't squished into 240px.
- **Save + phone→computer sync** (see above).
- **Live loop**: editing during a running loop (add/move beats, add/reorder tracks, mute/solo,
  swap instrument) is heard on the next loop pass — `fire()` reads live state each pass and
  refreshes lane nodes/gains. No stop/play needed.
- **Zoom controls** pinned to a fixed pill at the **bottom-right** (always visible).
- **Version in the title bar** (`<title>`), aligned with the SW cache version.

## Caveats

- The installed PWA's origin is fixed to whatever IP it was installed from; if the laptop's LAN
  IP changes, re-run `make-cert.sh` (and the phone reinstalls from the new address).
- Headless tests run muted — they verify scheduling/state logic, not literal sound.

## "My Sounds" sampler — PLAN (in progress)

Big feature: a **persistent, growing gallery of the user's own recorded sounds** ("My Sounds").
Each entry is an **editable audio sample** (waveform: trim/crop, length, base pitch, gain, EQ, loop)
that becomes a real **sampler instrument**. A recorded beatbox take is then used only to drive
**when/how** each hit plays: identity (which gallery sound), tempo/timing, intonation (pitch),
length, dynamics/mix. NOT trying to sound like the user — goal is right tone/tempo/intonation/length/mix.

Decomposition per hit: **identity** (gallery match) + **pitch** + **length** + **tone/EQ** (per-beat EQ
already built). Match on the **onset** (pitch-invariant) so "same sound longer/higher" still matches.
Sampler approach (chosen, avoids phase-vocoder): **pitch = resample (playbackRate)**, **length =
attack→sustain-loop→release envelope** (short hits truncate, long hits loop a middle region).

### Phases
- **Phase 1 — DONE (v0.5.0, 2026-06-28):** the gallery + waveform editor + **sampler voice**.
  Verified headless: add sound → assign to track (`type:'sample'`, `sound:'mys:<id>'`) → `_playLane`
  routes through `_samplePlay` with pitch-shift → persists across IndexedDB reload. Mic-record + the
  drag-to-trim waveform need real-device testing (headless can't use the mic).
  - IndexedDB store `mysounds` (bump `_idb` to v2; generalize `_idbTx(store,mode,fn)`).
  - State `mySounds:[]` (metadata only); raw audio cached in `this._soundRaw[id]`, AudioBuffers in `this._sampleAB[id]`.
  - `_samplePlay(id, at, vel, pitch, durSec, dest)`: resample for pitch (vs `basePitch`), trim offset/dur, loop to extend, gain+release env.
  - `_playLane` branch for `lane.type==='sample'` (lane.sound = `mys:<id>`). `_soundType(key)` → drum/synth/sample.
  - Picker: gallery entries appear in the unified `<select>` as "My — <name>"; `_soundName` resolves sample names.
  - Capture-to-gallery: `startSoundRec`/`_stopSoundRec` (mirror `_beginLaneCapture`/`_stopCapture`) → `_addSound`.
  - **My Sounds** modal + toolbar button: list (preview/rename/delete), record a sound, waveform editor
    (drag-to-trim, base-pitch, gain, loop toggle), Save.
- **Phase 2 — DONE (v0.6.0, 2026-06-28):** `_multiExtract` matches each detected onset → nearest gallery
  sound. Engine: `_fftMag` (radix-2 FFT) → `_segFeatures` (centroid/spread/rolloff/flatness/ZCR + 6 mel-ish
  bands) → `_matchSegment` (weighted k-NN, `MATCH_THRESH=0.62`; over threshold → "Unsorted" track). Matched
  groups become `type:'sample'` tracks. Toggle in My Sounds modal (`matchSounds`, default on). Verified
  headless: low hit→kick, high hit→hat, end-to-end split into correct sample tracks. **Listen-original:**
  every extracted hit stores `srcT`/`srcDur` into the take (`_origRec`); a track's Original/Extract toggle
  (now shown when `_laneCanOrig`) plays your real beatbox snippets via `_origVoice` in the playback paths.
  *Still TODO for Phase 2:* gallery sync to computer; "add unmatched to My Sounds" one-tap.
- **Phase 3:** per-hit pitch + length estimation from the take (intonation/length).
- **Later:** morphing/mutating sounds (trajectory, not a single label) — explicitly deferred.

Also pending (independent quick win): **live-recording onset markers under-count** vs final extract —
`_liveEvents` uses a coarser threshold than `_extract`; make them match so the take is trustworthy.

## Changelog

- **v0.7.6** — Minimap dot now reaches **every corner** of the grid. It used to represent the viewport
  *centre* (which can't be scrolled to an edge), so the "you are here" spot was pinned to screen-centre.
  The dot is now an **absolute grid position** (`_loc`): dragging places it exactly where you press and
  scrolls that point into view (X = timeline, Y = page), so it travels the full grid end-to-end
  (verified: top-left→bottom-right, ~0.98 range in both axes). When you're not dragging it, it tracks the
  centre of what's on screen (`_locFromView`, updated on both timeline and page scroll).
- **v0.7.5** — Replaced the minimap's blue viewport **square** with a single dot + a translucent
  "mirror" circle drawn on the REAL grid (see-through "you are here", shown only while the corner is open,
  follows the dot). Fixed the **blank-on-reopen** bug: the minimap canvas is remounted each time the corner
  opens, but the cached 2-D context still pointed at the old detached canvas → blank; now the context is
  re-acquired whenever the canvas element changes.
- **v0.7.4** — **Minimap is now an EXACT miniature of the whole grid** — one uniform scale (same for x
  and y) shrinks the entire grid to fit a ~248×190 corner box, so its proportions match the real grid
  exactly (verified: grid aspect ≈ minimap aspect). The location dot still drags freely in both axes
  (X pans the timeline, Y scrolls the tracks). Canvas is now sized explicitly (both width & height) instead
  of via CSS, which also removes the earlier "empty inside / drew before layout" glitch.
- **v0.7.3** — (1) **Track explosion fixed (the "54 tracks" disaster)** — every master take added a
  master lane + extracted lanes and never removed the previous take's, so re-recording piled up dozens of
  tracks, and a giant many-track canvas is what froze the machine (and broke the live markers, which redraw
  the whole grid each frame during capture). Auto-generated lanes are now tagged (`auto`/`srcMaster`);
  starting a new master take or re-extracting **replaces** the previous auto-split (hand-made/hand-recorded
  tracks are kept). Verified: extracting 3× stays at a bounded track count; a manual track survives.
  (2) **Minimap is now a tall 2-D navigation pad** — height grows with track count, and the location dot
  drags **freely in both axes** (X pans the timeline, Y scrolls the tracks/page). Also fixed the
  "empty inside / drew once then vanished" bug (it was drawn before layout → 0-width canvas; now retries via
  rAF). (3) **Extended sound, not just the block** — the per-hit "Original" length was capped at 0.8s
  (truncating long sounds); raised to 2.5s, and the matched sample auto-loops its sustain to fill the note
  (from v0.7.2), so a long "tsssss" actually sounds long.
- **v0.7.2** — (1) **Minimap location dot fixed** — dragging the dot used to move the cursor out of the
  corner's hover zone, firing `onMouseLeave`→`closeCorner` and unmounting the minimap mid-drag (felt like
  it "didn't work at all"). Now `onMinimapDown` sets `_cornerLock` and `closeCorner` ignores it during a
  drag (released on mouseup); the dot also centres on the visible span so it isn't pinned to an edge when
  the grid fits. (2) **Extended sounds now actually sound longer** — `_samplePlay` only looped the sample
  to fill a long note when the gallery sound had a *user-defined* loop region, so a long "tsssss" played a
  short "ts" then silence. It now auto-picks a sustain loop (short attack, loop the steady body) whenever
  the note is longer than the sample, so a short "ts" holds out into "tsssss". Extraction already captures
  the per-hit sustain as `len` (verified: a 0.45s burst → len≈0.43s vs 0.045s shorts).
- **v0.7.1** — (1) **Playback performance fix** — the moving red playhead now lives on its OWN
  viewport-sized overlay canvas (`playRef`/`_drawPlayhead`). During plain playback `_tick` redraws only
  that thin overlay (cheap, constant cost) instead of the whole timeline; the giant grid canvas is left
  untouched. This kills the stutter when zoomed in — verified: ~67 overlay draws vs only ~5 full grid
  redraws over 1s of playback on a 4120px-wide grid (was a full redraw EVERY frame). Recording still does
